Escalabilidad
01 · Contexto

¿Por qué ahora?

El Gateway como cuello de botella

Ya activamos los hilos virtuales en nuestros servicios de lógica, pero hoy vamos a aplicarlos donde más importa: el API Gateway.

El Problema

Al ser la puerta de entrada única, si el Gateway se queda sin hilos del sistema operativo, todo el sistema parece caído.

🔓

Al activar Project Loom, el Gateway dejará de usar hilos pesados y pasará a usar hilos virtuales

⚡

Permitiendo que miles de usuarios entren simultáneamente sin aumentar el consumo de memoria

También lo activaremos en Notification Service, para que el envío de mails (una tarea lenta y bloqueante) no detenga el procesamiento de otros mensajes.

10.1 / PASO 2
04 · Código

Paso 3: La Propiedad Mágica

Una línea de configuración con impacto masivo

En ambos servicios, dentro del bloque spring, agregaremos la llave mágica de escalabilidad: 

spring:
  threads:
    virtual:
      enabled: true  # Permite que Spring use Virtual Threads

📝 Nota importante: Al ser Spring Boot 4, esta propiedad configura automáticamente el servidor embebido para procesar cada petición en un hilo virtual independiente, evitando el cuello de botella tradicional de los hilos del SO.

10.1 / CÓDIGO
05 · Validación

Ejecución y Validación

Verificando el cambio

Para validar el cambio correctamente:

1

Reinicia el API Gateway

Observa que el arranque sigue siendo igual de rápido

2

Preparado para picos de tráfico

La JVM está ahora preparada para manejar picos que antes causaban "Too many threads"

3

Notification Service desbloqueado

El envío de correos vía SMTP no bloqueará el hilo que escucha RabbitMQ

✅ Sin cambios visibles: La belleza de los Virtual Threads es que son transparentes. El código no cambia, pero la capacidad de concurrencia se multiplica.

10.1 / VALIDACIÓN
06 · Resumen
Puntos Clave

Virtual Threads: Resumen

🚪

API Gateway: Puerta de entrada con capacidad ilimitada de concurrencia

📧

Notification Service: Envío de correos sin bloquear el consumo de eventos

⚡

Una línea de config: spring.threads.virtual.enabled: true

🔓

Impacto: Miles de peticiones simultáneas sin aumentar memoria

✨

Transparente: El código no cambia, la escalabilidad sí

Con esto, tu Gateway y tu sistema de notificaciones están preparados para escalar sin límites de hilos del sistema operativo.
